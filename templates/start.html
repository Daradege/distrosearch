The Kernel

The kernel is the core component of an operating system that acts as a bridge between applications and the actual data processing done at the hardware level. Key responsibilities include:

  • Memory management
  • Process management
  • Device drivers
  • System calls and security

The Linux kernel, created by Linus Torvalds in 1991, is monolithic (all core functions are part of the kernel) but supports loadable modules.

Unix Heritage

Unix was developed at Bell Labs in 1969 and became popular in academic and commercial environments. Its design philosophy emphasizes:

  • Small, modular programs that do one thing well
  • Plain text files for data storage
  • Hierarchical file system
  • Pipes and filters for connecting programs

Linux follows many Unix conventions but was written from scratch, making it Unix-like rather than a direct Unix descendant.

GNU and Linux Relationship

The GNU Project, launched in 1983 by Richard Stallman, aimed to create a complete Unix-compatible software system composed entirely of free software. By the early 1990s, GNU had produced many essential components but lacked a working kernel.

Linus Torvalds' Linux kernel filled this gap, leading to functional GNU/Linux systems. Today, most systems combine:

  • Linux kernel
  • GNU core utilities (ls, grep, bash, etc.)
  • X Window System or Wayland for graphics
  • A desktop environment
  • Other software components

What is a Linux Distribution?

A Linux distribution (distro) is an operating system made from a software collection based on the Linux kernel and often a package management system. Distros take the raw Linux kernel and combine it with:

  • System software and libraries from GNU
  • Package management system (APT, RPM, Pacman, etc.)
  • Desktop environment (GNOME, KDE, Xfce, etc.)
  • Additional software (web browsers, office suites, etc.)
  • Configuration tools and default settings

Unlike commercial OSes that have single versions (like Windows 10), Linux offers hundreds of distros catering to different needs.

Distribution Families

Most distros derive from one of these major families:

  • Debian-based (Ubuntu, Mint, Pop!_OS) - Known for stability and large repositories
  • Red Hat-based (Fedora, CentOS, RHEL) - Common in enterprise environments
  • Arch-based (Manjaro, EndeavourOS) - Rolling release with latest software
  • Slackware-based - One of the earliest distros, known for simplicity
  • Independent (Gentoo, Void) - Unique approaches to package management

Release Models

Distros follow different release strategies:

  • Fixed Release (Ubuntu, Fedora) - New versions released periodically with support periods
  • Rolling Release (Arch, openSUSE Tumbleweed) - Continuously updated with no version numbers
  • LTS (Long Term Support) - Versions with extended security updates (5+ years for Ubuntu LTS)

Linux Security Advantages

Linux is renowned for its security features, which stem from both technical design and community factors:

  • User Privileges: Regular users operate with limited permissions, preventing system-wide changes
  • Package Management: Software comes from trusted repositories with digital signatures
  • Open Source: Thousands of developers can review code for vulnerabilities
  • Rapid Updates: Security patches are available quickly, often within hours of discovery
  • SELinux/AppArmor: Mandatory access control systems restrict program capabilities
  • Smaller Target: Lower desktop market share means fewer malware attempts

Understanding sudo

The sudo (SuperUser DO) command allows permitted users to execute commands as the superuser or another user. Key aspects:

  • Unlike Windows' UAC, sudo requires explicit password entry (unless configured otherwise)
  • Commands are logged for accountability
  • Access is controlled via the /etc/sudoers file
  • Best practice is to use sudo for individual commands rather than working as root

Example usage: sudo apt update to refresh package lists

Linux in Other Systems

Linux's flexibility allows it to power many devices beyond traditional computers:

Android

Android uses a modified Linux kernel but replaces most userland components with Google's alternatives (Bionic libc, ART runtime, etc.). While technically Linux, it differs significantly from desktop Linux distributions.

ChromeOS

ChromeOS is Gentoo Linux with Google's customizations, running the Chrome browser as its primary interface. Recent versions can run Linux apps via a container.

Embedded Systems

Many routers, smart TVs, and IoT devices run Linux due to its customizability and lack of licensing fees.
